# オペランド「命令の実行結果」の具体例

オペランドには **変数・定数・命令の実行結果・リテラル** の4種類があります。  
このうち **「命令の実行結果」** とは、**メソッドを呼び出したときに返ってくる値**や、**式を計算した結果の値**が、そのまま別の演算子の「材料」として使われるものを指します。

---

## 1. メソッド呼び出しの戻り値がオペランドになる

**「命令」＝メソッドの呼び出し**  
**「実行結果」＝そのメソッドの戻り値（return で返る値）**

メソッド呼び出し `○○()` は**実行されると値に置き換わる**ので、その値が演算子のオペランドになります。

---

### 例1：`random.nextInt(3)` の戻り値が `+` のオペランド

```java
int recoverMp = second + new Random().nextInt(3);
//               ↑           ↑
//            変数        命令の実行結果（nextInt(3)の戻り値 0,1,2）
```

- `second` … **変数**（オペランド）
- `new Random().nextInt(3)` … **命令の実行結果**（オペランド）
  - `nextInt(3)` が実行される → 0, 1, 2 のいずれかが返る
  - その**返り値**が `+` の右側のオペランドになる

`nextInt(3)` という「命令」を実行した**結果の値**が、`+` の演算に使われています。

---

### 例2：`"hello".length()` の戻り値が `+` のオペランド

```java
int total = "hello".length() + 10;
//          ↑                   ↑
//    命令の実行結果           リテラル
//    (length()の戻り値 5)
```

- `"hello".length()` を実行 → `5` が返る
- その `5` が `+` の左のオペランド
- `10` が右のオペランド（リテラル）

`"hello"` や `10` を直接書く代わりに、**「length() を実行した結果の値」**がオペランドになっている例です。

---

### 例3：`Math.min(...)` の戻り値が `=` の右辺（オペランド）になる

```java
int recoverActual = Math.min(this.MAX_MP - this.mp, recoverMp);
//                   ↑
//            命令の実行結果（Math.min の戻り値）
```

- `Math.min(〇, △)` を実行 → 〇と△の小さい方の値が返る
- その**返り値**が `=` の右辺、つまり「代入される値」という意味でオペランド

代入の右辺も「演算の対象になる値」と広く考えれば、**Math.min の実行結果**がオペランドの一種だと言えます。

---

### 例4：`calculateSum(2, 3)` の戻り値がオペランド

```java
// calculateSum は「a + b を return する」メソッド
int x = calculateSum(2, 3) * 2;
//    ↑                      ↑
// 命令の実行結果(5)        リテラル
```

- `calculateSum(2, 3)` を実行 → `5` が返る
- その `5` が `*` の左のオペランド
- `2` が右のオペランド（リテラル）

「メソッドを実行した結果の値」が、掛け算のオペランドとして使われています。

---

## 2. 部分式の評価結果がオペランドになる

**「命令」＝一部分の式**  
**「実行結果」＝その式を計算した結果の値**

`(a + b)` や `this.MAX_MP - this.mp` のように、**先に計算される式**の結果も、外側の演算子にとっては「オペランド」です。

---

### 例5：`(a + b)` の評価結果が `*` のオペランド

```java
int a = 2, b = 3, c = 4;
int result = (a + b) * c;
//           ↑        ↑
//    命令の実行結果  変数
//    (a+b を計算した結果 5)
```

- まず `a + b` が実行される → `5` になる
- その `5` が `*` の左のオペランド
- `c` が右のオペランド

`(a + b)` という**式の実行結果**が、`*` のオペランドとして使われています。

---

### 例6：`this.MAX_MP - this.mp` の評価結果が `Math.min` のオペランド

```java
int recoverActual = Math.min(this.MAX_MP - this.mp, recoverMp);
//                          ↑                          ↑
//                   命令の実行結果                  変数
//                   (引き算の結果)
```

- `this.MAX_MP - this.mp` が先に計算される → ある整数になる
- その値が `Math.min` の**第1引数**として渡される
- メソッドの引数も「演算の対象」と考えると、ここでは **「引き算という式の実行結果」** がオペランドになっている、と解釈できます。

---

## 3. 一覧で整理

| オペランドの種類 | 例 | 役割 |
|------------------|-----|------|
| **変数** | `a`, `second`, `recoverMp` | 箱に入れた値 |
| **定数** | `MAX_HP`, `MAX_MP` | 変わらない値の箱 |
| **リテラル** | `3`, `"hello"`, `true` | ソースに直接書いた値 |
| **命令の実行結果** | `nextInt(3)` の戻り値、`(a+b)` の結果、`length()` の戻り値 | **メソッドや式を実行して得た値** |

---

## 4. イメージのまとめ

- **変数・定数** … すでに名前のついた「値の入れ物」
- **リテラル** … ソースコードに直接書いた `3` や `"こんにちは"` といった値
- **命令の実行結果** …  
  - メソッドを**実行したら**返ってくる値（`nextInt(3)` → 0,1,2 のどれか）  
  - または、式を**計算したら**出てくる値（`a + b` → 和）

「命令」＝メソッド呼び出しや一部分の式  
「実行結果」＝それらを実行・評価して得られた**値そのもの**  
その値が、`+` や `*` や `=` の右辺など、**演算子の対象**として使われるとき、それが「命令の実行結果」のオペランドです。

---

## 5. あなたのコードでの例（Cleric.java）

```java
int recoverMp = second + new Random().nextInt(3);
int recoverActual = Math.min(this.MAX_MP - this.mp, recoverMp);
```

- `second` … 変数（オペランド）
- `new Random().nextInt(3)` … **命令の実行結果**（0, 1, 2 のいずれかが返り、それが `+` の右オペランド）
- `this.MAX_MP - this.mp` … **式の実行結果**（その値が `Math.min` の第1引数＝オペランド）
- `recoverMp` … 変数（オペランド）
- `Math.min(..., recoverMp)` 全体の戻り値 … **命令の実行結果**（その値が `=` の右辺＝オペランド）

この一文のなかだけでも、「変数」「命令の実行結果」「リテラル（new や数値は別の文脈だが）」がオペランドとして使われている、と見ることができます。
