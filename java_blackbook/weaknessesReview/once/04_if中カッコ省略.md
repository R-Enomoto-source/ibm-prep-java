# 復習：if の中カッコ省略（問14）

## 間違えた理由

- 中カッコを省略したときに **「直後の 1 文だけ」が if に属する** というルールを適用していなかった。
- 「{} がなくてもコンパイルエラーにならない」は正しいが、**どの行までが if の対象か**を誤り、「A」「B」両方出る（C）やコンパイルエラー（E）を選んでしまった。
- 正しくは、**4 行目だけ**が if(false) の対象で、**5 行目は if の外**なので常に実行される。したがって「B」のみ表示（B が正解）。

---

## 重要なポイント

### 1. 中カッコ省略時のルール

**if の直後の 1 文だけ**が、条件が true のときに実行される処理になる。

- 2 行目、3 行目…と続いても、**if に属するのは直後の 1 文のみ**。
- それ以降の文は **if とは無関係**で、常に上から順に実行される。

### 2. 問14のコードで整理

```java
if (false)
    System.out.println("A");   // ← この 1 文だけが if に属する
System.out.println("B");       // ← if の外。常に実行される
```

- 条件は false なので、if に属する「A」の出力は実行されない。
- 「B」の出力は if の外なので **常に実行** → 結果は「B」のみ表示。

### 3. 読み方のコツ

- 中カッコが省略されているコードは、**頭の中で中カッコを補って考える**とよい。
- 上記は次と同じ意味：
  ```java
  if (false) {
      System.out.println("A");
  }
  System.out.println("B");
  ```

### 4. 試験対策

- if で {} を省略したときは、「**直後の 1 文だけ**」が if の本体。
- 見た目の「次の行」ではなく、**文の単位**で数える（1 行に複数文があれば、最初の 1 文だけが if に属する）。
- 実際の開発では、範囲を明確にするため中カッコを付けることが推奨される。
